# Shell Agent — システムアーキテクチャ

> ステータス: v0.7.0
> 日付: 2026-04-19

## 設計思想

shell-agentは**ローカルLLM推論**に最適化されたmacOS GUIチャット＆エージェントツールである。すべてのアーキテクチャ判断は、ユーザー自身のハードウェア上でLLMを実行する制約と機会に基づく:

- **クラウド非依存** — すべての知能はOpenAI互換API（LM Studio）経由でローカル実行
- **メモリ効率** — 3階層スライディングウィンドウがコンテキストオーバーフローを防止しつつ履歴を保持
- **ツール信頼性** — ReActではなくシンプルなフィードバックループ; テキスト内ツール呼び出しのためのGemmaタグパーサー
- **ユーザー制御** — 危険な操作にはMITL承認; ツール有効/無効トグル
- **データ主権** — すべてローカル保存: セッション、オブジェクト、ピン留め記憶、分析データベース

## コンポーネントマップ

### Goバックエンドパッケージ

| パッケージ | 責務 | 技術的特徴 |
|-----------|------|-----------|
| `app.go` | Wailsバインディング、ツールルーティング、ビジネスロジック | 統合ツールディスパッチ（シェル+MCP+内蔵） |
| `react.go` | エージェントループ、Gemmaタグパーサー、デバッグログ | Gemma-4ネイティブタグフォールバック |
| `internal/memory` | Hot/Warm/Cold階層、Pinned Memory、セッション | 時間認識スライディングウィンドウ+LLMコンパクション |
| `internal/client` | OpenAI互換APIクライアント（ストリーミング+非ストリーミング） | ラベル付きマルチモーダル画像対応 |
| `internal/toolcall` | シェルスクリプト登録、ジョブワークスペース、MITL | ヘッダーベース自動検出 |
| `internal/mcp` | mcp-guardian stdio、JSON-RPC 2.0 | マルチサーバー名前空間分離 |
| `internal/objstore` | 中央バイナリリポジトリ | 12文字hex ID、自動インデックス再構築 |
| `internal/analysis` | DuckDBエンジン、SQL生成、要約器 | 2層分析（インタラクティブ+バックグラウンド） |
| `internal/config` | JSON設定、パス展開 | ~/\$ENV展開 |

## データフロー: 1ターンの完全な流れ

```
1. ユーザーがメッセージ+画像（任意）を入力
   → ChatInput.tsx（IMEコンポジションガード、50ms WebKit遅延）
   → App.SendMessageWithImages(content, images[])

2. 画像をobjstoreに保存
   → 各data URL → objstore.SaveDataURL() → 12文字ID
   → ImageEntry{ID}参照のRecordを作成

3. システムプロンプト組立
   → 時刻 + タイムゾーン + 位置情報
   → Pinned Memory（英語）
   → Warm/Cold要約（時系列順）
   → Guardタグラッピング（ターンごとにノンスローテーション）

4. エージェントループ開始（react.go）
   → buildMessages()でセッションレコードからメッセージ構築
   → スマート画像処理: 最新はdata URL、古いものはテキスト参照
   → ツール定義付きでLLM呼び出し

5. レスポンス処理
   a. ツール呼び出しあり → 順次実行
      → write/executeカテゴリのMITLチェック
      → ツールごとにジョブワークスペース作成
      → アーティファクト → objstore
      → 結果 → メモリレコード
      → ループ継続（maxRoundsまで）

   b. テキストレスポンス → タイムスタンプ漏出を除去
      → メモリに保存
      → フロントエンドに返却

6. ターン後メンテナンス
   → トークン統計更新
   → 予算超過時メモリコンパクション
   → Pinned Memory抽出（バイリンガル）
   → セッション自動保存
```

## セキュリティアーキテクチャ

### 多層防御

| レイヤー | メカニズム | 目的 |
|---------|-----------|------|
| プロンプトインジェクション | nlk/guardノンスタグXML | データが命令として扱われることを防止 |
| JSON修復 | nlk/jsonfix | LLMの不正なJSON出力を修復 |
| 思考タグ | nlk/strip | モデルの思考プロセス漏出を除去 |
| ツール承認 | MITL（Man-In-The-Loop） | ユーザーが危険な操作を確認 |
| SQL制限 | プロンプト + DryRun | SELECTのみ、実行前に検証 |
| 画像分離 | objstore ID（data URLではない） | メモリ/ステートにbase64を含まない |

## 技術の集大成

shell-agentは以下の技術要素を統合したシステムである:

| 技術要素 | 実現している機能 |
|---------|---------------|
| **OpenAI互換APIクライアント** | ローカルLLM（LM Studio）によるチャット、ツール呼び出し、SQL生成、要約 |
| **MCP（mcp-guardian経由）** | 外部サービス連携のstdioプロキシ、複数サーバー名前空間分離 |
| **シェルスクリプトTool Calling** | ヘッダー自動検出、ジョブワークスペース、アーティファクト収集 |
| **内蔵コマンド** | 画像リコール、レポート生成、DuckDB分析（10ツール） |
| **時間認識メモリ** | タイムスタンプ注入、相対時間解決、Hot/Warm/Coldコンパクション |
| **空間認識** | タイムゾーン推論による位置情報、天気ツール連携 |
| **Pinned Memory** | バイリンガル自律的事実抽出、セッション横断永続化 |
| **マルチモーダル** | 画像入力、スマートリコール、レポート内ギャラリー |
| **セキュリティ** | nlk三位一体（guard/jsonfix/strip）、MITL承認、SQL制限 |
| **DuckDB分析** | 2層アーキテクチャ、スライドウィンドウ要約、バックグラウンドプロセス |

これらの要素が単一の.appバイナリに統合され、ローカル環境で完結する自律的なエージェントシステムを構成する。
